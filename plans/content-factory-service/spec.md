# Spec — Контент-завод для официального IG POWERELIX

> Статус: DRAFT (annotation cycle). Дата: 2026-06-10.
> Решения пользователя: развиваем `instagram-automation` в сервис; авто-постинг в свой IG (риск принят).

## Зачем (проблема)
Сейчас ведение официального Instagram POWERELIX — ручное и разрозненное: контент-план генерится CLI-скриптом, картинки — другим скриптом, постинг руками, разведки трендов нет, всё в файлах `output/`. Нет единого места, где идея превращается в опубликованный пост и где видно, что зашло.

## Что строим
Сервис на полке Штаба (FastAPI + SQLite + HTMX, как wb-promotion), который проводит пост по конвейеру:
**Разведка → Контент-план → Банк идей → Генерация → Аппрув → Публикация → Аналитика.**

Развиваем существующий репо `instagram-automation` (переиспользуем рабочий контент-план и Grok-генерацию), а не строим с нуля и не трогаем `content-factory` (тот — отдельный блогерский трек, в нём IG-постинг намеренно запрещён).

## Не входит в MVP (явно)
- Блогерская UGC-CRM (это `content-factory`, отдельно).
- Постинг в VK/Telegram/Rutube (только свой IG; мульти-канал — потом).
- Масс-директ, ответы на комменты, Stories-автоматизация (фаза «потом»).
- Видео-генерация Reels (MVP — фото/карусели; Reels-видео — отдельная фаза, дорого).

## User stories + acceptance criteria

### US-0. Разведка трендов
Как маркетолог, хочу видеть вирусные чужие Reels по нашим темам с разбором хуков, чтобы черпать идеи.
- AC-0.1: можно запустить парсинг по теме/хэштегу (витамины, ЗОЖ, похудение) → в БД появляются ролики с просмотрами, лайками, ссылкой, текстом.
- AC-0.2: каждый собранный ролик можно прогнать через разбор (Claude): хук, приём удержания, эмоц-триггер, структура, «почему зашло», адаптированная идея под POWERELIX.
- AC-0.3: ролики сортируются/фильтруются по просмотрам; разобранная идея одной кнопкой улетает в Банк идей.
- AC-0.4: mp4-ссылки Apify живут ~3 дня — видео (или кадр) сохраняется сразу при сборе.

### US-1. Контент-план
Как маркетолог, хочу сгенерировать план на период с балансом рубрик.
- AC-1.1: задаю N постов, старт, частоту, опц. фокус → Claude отдаёт план (переиспользуем `content_plan.generate`).
- AC-1.2: план и его посты сохраняются в БД (не только в файл).
- AC-1.3: посты плана одной кнопкой создаются как черновики в очереди постов.

### US-2. Банк идей и хуков
Как маркетолог, хочу копилку идей из разведки/плана/вручную.
- AC-2.1: идея имеет источник (тренд/план/вручную), рубрику, продукт, статус (новая/в работе/использована).
- AC-2.2: из идеи можно создать черновик поста.

### US-3. Генерация поста
Как маркетолог, хочу из черновика получить готовый визуал + текст.
- AC-3.1: визуал генерится через `scenes.generate_scene` с постоянным лицом AI-модели бренда (`assets/brand/ai_model.png`).
- AC-3.2: текст (подпись, хэштеги, хук) генерится Claude (или берётся из плана).
- AC-3.3: сгенерированные ассеты привязаны к посту, видны в карточке, можно перегенерировать.

### US-4. Аппрув + БАД-комплаенс
Как владелец, хочу проверить пост перед публикацией и не нарушить закон о рекламе БАД.
- AC-4.1: пост проходит линт стоп-слов («лечит/избавляет/диагноз…») — нарушения подсвечены.
- AC-4.2: чек наличия дисклеймера «не является лекарственным средством» для продуктовых постов.
- AC-4.3: пост нельзя перевести в «одобрено», пока не пройдены чеки (или явный override владельцем с пометкой).

### US-5. Планирование и публикация
Как маркетолог, хочу запланировать пост и чтобы он опубликовался сам в нужное время.
- AC-5.1: у одобренного поста задаётся дата/время публикации.
- AC-5.2: планировщик (APScheduler) в нужный момент публикует пост в IG через Graph API (контейнер → media_publish), для фото и каруселей.
- AC-5.3: публикация идемпотентна — пост не публикуется дважды; результат (ig_media_id, permalink) пишется в БД; ошибка → статус failed + причина.
- AC-5.4: токен IG автоматически продлевается (cron) до истечения 60 дней; при невалидном токене публикация не молча падает, а помечается и алертит.

### US-6. Аналитика
Как владелец, хочу видеть, что зашло.
- AC-6.1: для опубликованных постов тянутся метрики (reach, likes, comments, saves, shares, plays) через insights.
- AC-6.2: видно сводку: какие рубрики/хуки дают охват.

## Внешние зависимости и риски (вне нашего кода)
- **[BLOCKER на реальный постинг]** Content Publishing API требует **Instagram Business + App Review Meta** на `instagram_business_content_publish` (~2–4 недели). Без этого код постинга писать можно, но публиковать нельзя. → MVP: код готов, реальная публикация включается после ревью. Пока — режим «черновик/симуляция».
- **[NEEDS CLARIFICATION]** Подключённый сейчас аккаунт — Business или Creator? (Creator через API публиковать НЕЛЬЗЯ.)
- **[NEEDS CLARIFICATION]** Claude через ProxyAPI: `content_plan.generate` использует `client.messages.parse(output_format=…, thinking=…)` — поддерживает ли это наш ProxyAPI-эндпоинт, или нужен прямой Anthropic-ключ для structured output? (память: на проде Haiku 4.5 через ProxyAPI ради экономии.)
- **[ЮР]** Авто-постинг в IG — серая зона (риск принят пользователем). Снижаем риск: только органика бренда, БАД-дисклеймеры, без явных «реклама»-форматов.
- Apify-акторы периодически ломаются, выборка по хэштегам неполная, mp4-ссылки протухают ~3 дня.
- IG-лимит 100 публикаций/24ч (для нас неактуально по объёму, но заложить проверку).

## Где живёт / деплой
Новый systemd-сервис на VPS (рядом с wb-promotion, отдельный порт), nginx-поддомен (напр. `content.bandabogachey.online`), плитка на Штабе. Деплой — финальная фаза.
