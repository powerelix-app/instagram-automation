# Референсы по ссылке: Pinterest (+ развитие до «стиль-памяти»)

Дата: 2026-07-14. Статус: MVP в работе.

## Идея (от пользователя)

«Кидаю ссылку на понравившийся контент → завод считывает дизайн, запоминает,
генерирует такое же под наш товар». Источники: Pinterest-пины, IG-карусели,
IG-Reels, IG-аккаунты целиком.

## Ресёрч: что уже есть

Пайплайн «референс → разбор → сделать похожий → пост» ПОСТРОЕН в Разведке:

- `recon.add_reel_by_url(url)` — IG-ссылка → TrendReel (video/carousel/image)
- `recon.deep_analyze_images` / `deep_analyze` — Claude vision разбор
  (хук, структура, дизайн/вёрстка в `camera_work`)
- `recon.make_similar(analysis_id, product_id)` — сцены 1-в-1 по слайдам
  референса, но с нашим продуктом → Storyboard → пост → аппрув → публикация
- Разведка по аккаунтам: `scrape_account(@handle)` — уже есть

Не хватает ТОЛЬКО приёма Pinterest-ссылок. IG-часть閉 замкнута.

## Ресёрч: как достать картинку из Pinterest без API-ключей

1. `GET https://widgets.pinterest.com/v3/pidgets/pins/info/?pin_ids=<id>` —
   публичный виджет-эндпоинт, JSON с `images` (236x/564x), `description`,
   board, pinner. Проверено 2026-07-14: работает без auth с Мака.
2. Апгрейд размера: у i.pinimg.com URL меняем сегмент `/564x/` → `/originals/`
   (частый паттерн), при 404 фолбэк `/736x/`, затем 564x.
3. `pin.it` короткие ссылки — редирект на полный URL (follow redirects).
4. Риск: РКН/гео на VPS — эндпоинт может быть недоступен с РФ-VPS.
   Митигируется `_fetch_bytes`-фолбэком (apify fetch_via_actor) и тем,
   что добавление идёт через браузер пользователя → сервер; если widgets
   недоступен именно с VPS — добавить relay позже (как tg/ig).

## Решение MVP (минимальный дифф)

Расширить существующий вход «по ссылке» в Разведке:

- `apify.py` или `recon.py`: `_pin_by_url(url)` — распознать
  `pinterest.*/pin/<id>` и `pin.it/<code>`, дёрнуть widgets API,
  вернуть норм-структуру как у `reel_by_url` (media_type="image",
  images=[best_url], caption=description, username=pinner).
- `recon.add_reel_by_url`: если URL пинтерестовский → ветка `_pin_by_url`,
  topic="pinterest" (свой чип в Разведке).
- Дальше НИЧЕГО нового: existing `_ensure_images` скачает файл,
  `deep_analyze_images` разберёт дизайн, `make_similar` соберёт ремейк
  под выбранный товар, storyboard → пост → ВК/ТГ/IG.

UI: поле «Добавить по ссылке» уже есть на /recon — работает и для Pinterest.

## Критика («10 причин обосраться», выжимка)

1. widgets API отдаёт максимум 564x? — берём originals-апгрейд, фолбэк.
2. Видео-пины — MVP: сохраняем постер (images), пометка в caption.
3. РКН на VPS — см. митигацию выше; проверить вживую при деплое.
4. Дедуп — по норм-URL пина (как для IG).
5. `deep_analyze_images` заточен под карусели IG — промпт универсален
   (слайды по порядку), один слайд = частный случай. ОК.
6. Не ломаем IG-ветку: распознавание Pinterest ДО вызова apify.
7. pin.it редиректы — requests follow_redirects, достаём id из финального URL.
8. Секретов не добавляем; наружу шлём только id пина. Инъекции нет
   (id валидируется как \d+).
9. Лимиты Pinterest — добавление ручное/штучное, батчей нет.
10. Стиль-«память»: пока = записи в Разведке с разбором (фильтр по чипу
    pinterest). Отдельная библиотека стиль-профилей — фаза 2.

## Фазы

1. **MVP (сейчас)**: Pinterest-пин по ссылке → Разведка → разбор → ремейк.
2. Библиотека стиль-профилей: сохранённые разборы как переиспользуемые
   «стили» (выбор стиля при генерации любого поста).
3. Pinterest-доски и IG-аккаунты как источник стиля бренда (сетка → профиль).
4. Reels-ремейк из Pinterest-видео (постер уже есть, видео — позже).
